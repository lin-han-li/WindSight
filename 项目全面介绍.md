# WindSight（智风监测系统）项目全面介绍

WindSight 是一个面向风电/工业现场的**轻量级数据采集、实时监测与历史回放**系统。项目以 **Flask + Flask‑SocketIO + SQLAlchemy（SQLite）** 为后端核心，使用 **Jinja 模板 + 原生 JavaScript + Bootstrap + ECharts** 提供可视化界面，支持终端节点通过 HTTP 上报数据、服务端入库并通过 WebSocket 推送到浏览器，实现“实时看、历史查、数据管”的闭环。

---

## 核心能力一览

- **节点数据上报（HTTP）**
  - 接口：`POST /api/upload`
  - 支持节点按时间持续上报，后端持久化存储（默认 SQLite）
- **实时监测（WebSocket / Socket.IO）**
  - 服务端推送：`monitor_update` / `node_data_update`
  - 页面无刷新实时追加与渲染
- **历史回放与数据概览**
  - 按节点/通道查看历史波形
  - 支持缩放、平移（Grab 模式）等交互
- **系统设置 / 数据管理**
  - 按保留天数清理历史数据
  - 按节点删除数据、清空数据（高危）
  - **SQLite VACUUM**：释放磁盘空间（SQLite 删除后文件不变，需要 VACUUM 才能收缩）
- **模拟器**
  - `sim.py` 可生成并上报模拟数据，方便联调与演示
- **登录认证**
  - 基于 `Flask-Login` 的登录/登出
  - 支持“首次启动自动创建默认管理员”

---

## 技术栈

- **后端**
  - Python 3.11
  - Flask 3.x
  - Flask-SocketIO 5.x（实时推送）
  - SQLAlchemy（通过 Flask-SQLAlchemy 3.x）
  - eventlet / gevent（SocketIO 异步模式）
  - gunicorn（生产部署）
- **前端**
  - HTML5 + Jinja2 模板
  - 原生 JavaScript
  - Bootstrap 5（UI）
  - ECharts 5（曲线/图表）
- **数据库**
  - 默认 SQLite（`database/wind_farm.db`）

---

## 系统架构（数据流）

1. **终端节点**以固定频率向后端调用 `POST /api/upload` 上报一帧数据  
2. 后端对数据进行**校验/归一化**后写入数据库（`node_data` 表）  
3. 后端通过 Socket.IO 将最新数据推送给订阅该节点的前端页面  
4. 前端实时追加曲线；历史页按时间范围拉取数据，绘制回放曲线

## 目录结构与模块说明

> 代码主要位于 `code/` 目录；数据库默认位于 `database/`。

- `code/app.py`
  - 应用入口：加载环境变量、初始化 Flask/SocketIO、注册蓝图、数据库初始化、默认管理员创建等
- `code/windsight/`
  - `config.py`：配置项（`DATABASE_URL`、CORS、日志、SocketIO 参数、密码策略等）
  - `models.py`：数据库模型
    - `User`：登录用户
    - `SystemConfig`：系统配置（JSON 文本）
    - `NodeData`：上报数据（核心表，32 通道数据以 JSON 字符串存储）
  - `routes/`
    - `api.py`：数据上报/查询/清理/VACUUM 等 API
    - `auth.py`：登录/登出
    - `pages.py`：页面路由（overview/monitor/settings 等）
  - `socket_events.py`：Socket.IO 事件与订阅管理
  - `time_utils.py`：时间格式化与时区相关工具
- `code/templates/`
  - Jinja 页面模板（`base.html`、`login.html`、`monitor.html`、`overview.html`、`settings.html` 等）
- `code/static/js/`
  - 前端图表与交互逻辑（实时订阅、回放、缩放/平移等）
- `code/sim.py`
  - 数据模拟器：生成/上报测试数据
- `database/`
  - 默认 SQLite 数据库位置（项目运行会创建/写入）

## 配置与环境变量（重点）

项目启动时会按优先级加载环境变量文件：

1. `WINDSIGHT_ENV_FILE` 指定的文件
2. `code/windsight.env`（推荐）
3. `code/.env`（可选）

模板配置文件：

- `code/env.example`
- `code/windsight.env.example`

常用配置项：

- **`SECRET_KEY`**：生产环境必须设置，避免重启后会话失效
- **`DATABASE_URL`**：数据库连接串（默认 `sqlite:///../database/wind_farm.db`）
- **默认管理员**
  - `WINDSIGHT_DEFAULT_ADMIN_ENABLED=1`
  - `WINDSIGHT_DEFAULT_ADMIN_USERNAME=WindSight`
  - `WINDSIGHT_DEFAULT_ADMIN_PASSWORD=...`
    - 若不设置，首次启动会生成随机密码并写入日志（更安全）
- **CORS**
  - `ALLOWED_ORIGINS=*`（开发方便；生产建议限制域名）
- **密码策略**
  - `PASSWORD_MIN_LENGTH` / `PASSWORD_REQUIRE_UPPERCASE` / `PASSWORD_REQUIRE_DIGITS` / `PASSWORD_REQUIRE_SPECIAL`

## 本地运行（Windows / 开发）

1. 创建虚拟环境并安装依赖：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r code\requirements.txt
```

2. 配置环境变量文件：

```bash
copy code\env.example code\windsight.env
```

3. 启动：

```bash
cd code
python app.py
```

浏览器访问：`http://localhost:5000`

## 生产部署建议（阿里云/Ubuntu 常见做法）

推荐组合：**gunicorn（eventlet worker） + nginx 反代 + systemd 守护**。

部署要点：

- 应用只监听 `127.0.0.1:5000`，由 Nginx 对外提供 `80/443`
- Socket.IO 需要 Nginx 设置 `Upgrade/Connection` 以支持 WebSocket
- SQLite 建议单进程/单 worker（如需多进程/多机，请引入 Redis 等消息队列）

## 数据库与空间管理（SQLite）

为什么删了数据文件大小不变？

- SQLite 的 `DELETE` 只标记空间可复用，不会收缩文件
- 需要执行 **VACUUM** 才会物理收缩数据库文件

本项目提供：

- 设置页按钮：**压缩数据库（VACUUM）**
- 管理接口：`POST /api/admin/vacuum`

## 常见问题（FAQ）

- **忘记管理员密码怎么办？**
  - 密码加密存储无法直接查看
  - 可使用 `code/reset_password.py` 重置指定用户密码：
    - `python reset_password.py WindSight MyNewPassword123`
- **页面改了刷新没变化？**
  - 可启用模板热更新开关：`WINDSIGHT_TEMPLATE_AUTO_RELOAD=1`
- **Windows 下 eventlet 报错/兼容问题？**
  - `app.py` 支持 `FORCE_ASYNC_MODE=auto|eventlet|gevent|threading`
  - Windows/新 Python 版本建议使用 Python 3.11 并优先 `threading` 或 `gevent`

## 安全与运维建议

- **不要提交敏感文件**：`windsight.env`、`.env`、数据库文件、日志文件
- 生产环境建议：
  - 强随机 `SECRET_KEY`
  - 限制 `ALLOWED_ORIGINS`
  - 通过 Nginx + HTTPS 对外提供服务
  - 定期备份 `database/wind_farm.db`
